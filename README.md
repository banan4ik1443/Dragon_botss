# 🐉 Dragon Bot — MVP

Рабочий каркас telegram-бота про драконов на **aiogram 3** + **SQLite (aiosqlite)**.
Реализован полный основной цикл: обучение → яйцо → вылупление → кормление/тренировка →
усталость → дуэли → клыки → магазин/скупка. Часть фич из дизайн-документа — заглушки
или не реализованы, см. таблицу ниже.

## Запуск

```bash
cd dragon_bot
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env          # вписать туда токен от @BotFather
python main.py
```

База данных создаётся автоматически (`dragon_bot.db`, SQLite) при первом запуске.

## Структура проекта

```
dragon_bot/
├── main.py            # точка входа, подключение роутеров
├── config.py           # токен, таймзона МСК, время обновления магазина
├── game_data.py         # редкости, виды драконов, способности, экономика (числа)
├── game_logic.py         # чистые формулы: усталость, статы, клыки, цена продажи
├── database.py           # SQLite-схема и CRUD
└── handlers/
    ├── start.py          # /start, онбординг, главное меню, помощь
    ├── profile.py          # профиль, смена ника, настройки
    ├── dragon.py            # список драконов, кормление, тренировка, выбор активного
    ├── eggs.py                # отдельное меню "Открытие яиц" с анимацией раскрытия
    ├── explore.py            # локации → яйца/мясо
    ├── shop.py                # броня, акции дня, скупка драконов
    ├── pvp.py                  # дуэли, очередь матчмейкинга, расчёт клыков
    ├── quests.py                # ежедневные задания, стрик, награды
    └── top.py                    # топ сезона по клыкам, сброс сезона, награды
```

## Как теперь устроены яйца и рост дракона

Раньше яйцо вылуплялось по таймеру (дольше для редких). По твоей просьбе
переделано: **яйцо открывается сразу**, с короткой анимацией раскрытия
(`handlers/eggs.py`, несколько edit_text с задержкой ~0.7с и нарастающими
эффектами по редкости). Дракон появляется мгновенно.

"Время ожидания" теперь не на яйце, а на самом драконе: у каждой редкости
есть `min_duel_level` (`game_data.RARITIES`) — минимальный уровень, на
котором дракон допускается до дуэлей:

| Редкость | Уровень для дуэлей |
|---|---|
| Обычный | 3 |
| Необычный | 4 |
| Редкий | 5 |
| Эпический | 6 |
| Легендарный | 8 |
| Мифический | 10 |

Чем реже дракон — тем дольше его растить кормлением/тренировками перед
тем, как пустить в бой. Раздел «🐉 Драконы» показывает только настоящих
драконов (🌱 — ещё растёт, ⚔️ — готов к дуэлям), яйца переехали в
отдельное меню «🥚 Открытие яиц».

## Оформление

Карточки профиля/дракона/заданий/топа используют Telegram-цитирование
(`<blockquote>...</blockquote>`) для блока характеристик и разделитель
«»» вместо «:» — под стиль, который ты прислал на скрине.

## Что реализовано полностью

- Онбординг с историей, стартовое яйцо + ресурсы
- Открытие яиц с анимацией, дракон выдаётся сразу; "ожидание" перенесено
  на рост дракона до `min_duel_level` нужной редкости (см. раздел выше)
- Кормление и тренировка дракона с расходом мяса/монет, начислением опыта
- **Усталость**: до 50% статы полные, 50–100% — линейное падение статов
  (`fatigue_stat_multiplier`), на 100% — полный бан действий. Восстанавливается
  естественно со временем (`FATIGUE_DECAY_PER_HOUR`)
- Дуэли: матчмейкинг через очередь, расчёт клыков по твоим правилам
  (буст новичкам первые 3 дня, буст в выходные, защита от фарма слабых
  соперников, влияние "онлайна" на разброс) — см. `calculate_fangs_change`
- Локации (`/explore`) — источник яиц/мяса с кулдауном
- Магазин: броня (всегда), ежедневная ротация акций (детерминированно
  по дате, обновление считается от 11:00 МСК), скупка дракона по формуле
  редкость+уровень+способность
- Профиль с нужными полями
- **Ежедневные задания** (`/quests`, алиас `/daily`): всегда включают
  "зайти в бота", плюс до 2 случайных из {исследовать, покормить,
  потренировать, сыграть дуэль} — причём заданию "покормить/потренировать/
  сыграть дуэль" нужен активный дракон, так что новичку без дракона их
  просто не выдадут (только "зайти" + "исследовать")
- **Стрик заданий**: считается за полное выполнение ВСЕХ заданий дня
  (отдельный "бонусный" пункт автозавершается, когда выполнены остальные).
  Пропуск дня сбрасывает стрик. Чем длиннее стрик — тем выше награды за
  все задания (до +112% при капе в 14 дней, см. `quest_streak_multiplier`)
- **Топ сезона** (`/top`): живой рейтинг по клыкам, сезон = календарный
  месяц по МСК. При смене месяца — ленивый сброс: снимок топ-20 уходит в
  историю с наградами по местам, клыки у всех обнуляются. Награды можно
  забрать кнопкой "Награды сезона" (хранятся, пока не заберёшь — сезон
  не потеряется, если зайти позже)

## Что сделано как заглушка / не реализовано

Эти разделы упомянуты в дизайн-документе, но требуют отдельной проработки
(БД-схему под них закладывать тоже отдельно):

| Фича | Статус |
|---|---|
| Донат за Stars, VIP-статусы | кнопка-заглушка, нужна интеграция Telegram Stars Invoice API |
| Батл-пасс | не реализован, нужна модель сезона + трек наград |
| Реальный "онлайн" для PvP-формулы | `_online_factor()` в `pvp.py` — заглушка на размере очереди, нужно подключить честную метрику активных игроков |
| Ивенты (Гроза и т.п.) | не реализованы, нужен APScheduler с окном пауз 22:00–06:00 МСК |
| Сброс сезона по реальному времени | сейчас сброс топа "ленивый" — срабатывает при первом заходе кого-то в `/top` после смены месяца, а не ровно в 00:00; для точного времени нужен APScheduler |
| Механика "отдых X часов" отдельно от естественного восстановления | сейчас усталость просто стекает по времени (~25 часов с 100 до 0); поля `is_resting`/`rest_until` в БД заведены, но не используются — это для фичи "ускорить отдых за звёзды" |

## Известные допущения по балансу

Все числа в `game_data.py` (стоимость кормления/тренировки, прирост опыта,
пороги усталости, цены брони) — черновые, подбираются на плейтестах.
Это сделано намеренно отдельным файлом, чтобы баланс правился без касания
логики бота.
